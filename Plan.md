# План автотестирования

## 1. Перечень автоматизируемых сценариев

### Позитивные сценарии:
1. **Оплата по дебетовой карте** (APPROVED)
   - Переход на страницу оплаты
   - Заполнение формы картой `4444 4444 4444 4441`
   - Проверка уведомления об успешной оплате
   - Проверка статуса в БД (`APPROVED`)

2. **Оплата в кредит** (APPROVED)
   - Переход на страницу оплаты через кнопку "Купить в кредит"
   - Заполнение формы картой `4444 4444 4444 4441`
   - Проверка уведомления об успешной оплате
   - Проверка статуса в БД (`APPROVED`)

### Негативные сценарии:
1. **Оплата по дебетовой карте** (DECLINED)
   - Переход на страницу оплаты
   - Заполнение формы картой `4444 4444 4444 4442`
   - Ожидаемое сообщение об ошибке
   - Проверка статуса в БД (`DECLINED`)

2. **Оплата в кредит** (DECLINED)
   - Переход на страницу оплаты через кнопку "Купить в кредит"
   - Заполнение формы картой `4444 4444 4444 4442`
   - Ожидаемое сообщение об ошибке
   - Проверка статуса в БД (`DECLINED`)

3. **Пустые поля формы**
   - Отправка формы без заполнения
   - Ожидаемое сообщение о необходимости заполнить поля

4. **Неверный формат номера карты**
   - Ввод номера с буквами или недостаточным количеством цифр
   - Ожидаемая ошибка валидации

5. **Просроченная карта**
   - Ввод года меньше текущего
   - Ожидаемая ошибка

## 2. Инструменты

| Инструмент | Назначение | Обоснование |
|---|---|---|
| Python 3.14+ | Язык написания тестов | Популярный, простой, много библиотек |
| pytest | Фреймворк для тестов | Удобный, гибкий, интеграция с Allure |
| Selenium WebDriver | UI-тесты | Стандарт для автоматизации браузера |
| requests | API-тесты | Для проверки gate-simulator (опционально) |
| pymysql | Проверка БД (MySQL) | Для подтверждения сохранения статуса платежа |
| Allure | Отчёты | Наглядная визуализация результатов |
| Docker | Контейнеризация | Для изоляции и запуска тестовой среды |
| GitHub Actions | CI | Автоматический запуск тестов при пуше |

## 3. Риски

| Риск | Вероятность | Способ снижения |
|---|---|---|
| Проблемы с Docker | Средняя | Подробная документация, проверка локально |
| Нестабильные тесты (flaky) | Средняя | Использовать явные ожидания (WebDriverWait) |
| Баги в приложении | Высокая | При падении тестов из-за багов приложения - фиксировать дефекты в виде Issue |
| Сложность с SQL-запросами | Средняя | Проверить структуру БД заранее |
